# 演示手册

## 一、演示目标

本系统当前适合演示 2 条核心业务闭环：

1. 入社申请闭环
2. 活动管理闭环

同时可以展示：

- 公告发布与查看
- 成员管理
- 多角色登录

## 二、演示前准备

请先确认：

- MySQL 已导入初始化脚本
- 后端已启动
- 前端已启动
- 能正常访问登录页

推荐准备 3 个账号：

- 系统管理员：`superadmin / 123456`
- 社团管理员：`admin / 123456`
- 学生用户：`user / 123456`

## 三、演示路径一：入社申请闭环

### 步骤 1：学生登录

使用学生账号登录：

```text
user / 123456
```

进入：

- 社团列表 `/clubs`

### 步骤 2：学生申请加入社团

- 选择一个尚未加入的社团
- 点击“申请加入”
- 再进入“入社申请”页面 `/join-applications`
- 可以看到自己的申请状态为“待审核”

### 步骤 3：管理员登录审批

退出当前账号，使用社团管理员登录：

```text
admin / 123456
```

进入：

- 入社申请 `/join-applications`

操作：

- 查看待审批申请
- 点击“通过”

### 步骤 4：验证审批结果

重新用学生账号登录，进入：

- 入社申请 `/join-applications`

可以看到申请状态已变为：

- `已通过`

说明入社申请闭环完成。

## 四、演示路径二：活动管理闭环

### 步骤 1：管理员发布活动

使用社团管理员账号登录，进入：

- 活动管理 `/activities`

点击：

- “发布活动”

填写：

- 所属社团
- 活动名称
- 活动简介
- 活动详情
- 活动要求
- 活动时间

提交后活动会出现在活动列表中。

### 步骤 2：学生报名活动

切换到学生账号登录，进入：

- 活动管理 `/activities`

操作：

- 查看活动列表
- 选择一个活动点击“报名参加”

### 步骤 3：管理员查看报名名单

切回管理员账号，进入：

- 活动管理 `/activities`

找到对应活动，点击：

- “查看名单”

可以看到报名人员信息：

- 姓名
- 性别
- 电话
- 报名时间

说明活动管理闭环完成。

## 五、演示路径三：公告与成员管理

### 公告中心

管理员登录后进入：

- 公告中心 `/notices`

可以：

- 发布社团公告
- 查看已有公告

系统管理员登录后：

- 可以发布系统公告

### 成员管理

管理员登录后进入：

- 成员管理 `/members`

可以：

- 查看成员列表
- 移除普通成员

限制：

- 不能移除社团管理员

## 六、推荐答辩展示顺序

建议按以下顺序展示：

1. 先展示登录页与角色区别
2. 展示学生端社团申请
3. 展示管理员审批申请
4. 展示管理员发布活动
5. 展示学生报名活动
6. 展示管理员查看报名名单
7. 补充展示公告和成员管理
