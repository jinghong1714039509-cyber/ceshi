# 启动说明

## 一、环境要求

### 后端

- JDK 17
- Maven 3.8 及以上
- MySQL 8

### 前端

- Node.js 22 及以上
- npm 10 及以上

## 二、数据库初始化

### 1. 创建数据库

建议先创建数据库：

```sql
CREATE DATABASE association_manager DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

### 2. 导入初始化脚本

先执行：

- [`db/init.sql`](/C:/Users/Administrator/Desktop/ceshi/db/init.sql)

再执行：

- [`db/patches/001_bootstrap_security.sql`](/C:/Users/Administrator/Desktop/ceshi/db/patches/001_bootstrap_security.sql)

## 三、后端启动

进入目录：

- [`backend`](/C:/Users/Administrator/Desktop/ceshi/backend)

执行命令：

```powershell
mvn spring-boot:run
```

默认配置：

- 端口：`8080`
- 接口前缀：`/api`

数据库配置文件位置：

- [`backend/src/main/resources/application.yml`](/C:/Users/Administrator/Desktop/ceshi/backend/src/main/resources/application.yml)

### 数据库配置方式

后端支持两种方式配置数据库：

1. 直接修改 `application.yml`
2. 通过环境变量覆盖

可用环境变量：

- `DB_URL`
- `DB_USERNAME`
- `DB_PASSWORD`

PowerShell 示例：

```powershell
$env:DB_URL="jdbc:mysql://localhost:3306/association_manager?zeroDateTimeBehavior=convertToNull&serverTimezone=Asia/Shanghai&useUnicode=true&characterEncoding=utf8&useSSL=false&allowPublicKeyRetrieval=true"
$env:DB_USERNAME="root"
$env:DB_PASSWORD="你的数据库密码"
mvn spring-boot:run
```

如果数据库账号密码不正确，接口现在会直接返回更明确的提示：

```text
数据库连接或查询失败，请检查 application.yml 或环境变量中的数据库配置
```

## 八、常见问题

### 1. 登录时提示“数据库连接或查询失败”

优先检查：

- MySQL 服务是否启动
- 数据库 `association_manager` 是否已创建
- 是否已经导入 `db/init.sql`
- 是否已经执行 `db/patches/001_bootstrap_security.sql`
- `DB_USERNAME` 和 `DB_PASSWORD` 是否与本机 MySQL 一致

### 2. 页面能打开但接口失败

优先检查：

- 后端是否运行在 `8080`
- 前端代理是否仍指向 `http://localhost:8080/api`
- 浏览器开发者工具里是否返回 `401` 或 `500`

### 3. 前端构建时提示 chunk 过大

这是当前版本已知问题，不影响开发和演示。后续可通过路由懒加载和组件按需引入继续优化。
